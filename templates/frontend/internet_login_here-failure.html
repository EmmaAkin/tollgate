There was a glitch in the matrix, making it impossible for me to get your MAC address.

Without this, I cannot log you in. Please write down your MAC address and IP address and see an admin.

Getting your MAC Address in Windows NT, 2000, XP, 2003, Vista, 2008

  1. Start → Run → cmd → OK
  2. Type ipconfig /all
  3. Copy down your IP address and Physical Address (MAC)

Getting your MAC Address in Linux

  1. Open a Terminal
  2. Type /sbin/ifconfig | grep -A1 HWaddr
  3. Copy down your IP address (inet addr) and MAC Address (HWaddr)

Example

$ /sbin/ifconfig | grep -A1 HWaddr
eth0      Link encap:Ethernet  HWaddr 00:18:XX:XX:XX:XX
          inet addr:10.13.XX.XXX  Bcast:10.13.3.255  Mask:255.255.240.0
--
vmnet1    Link encap:Ethernet  HWaddr 00:50:56:C0:00:01  
          inet addr:192.168.10.1  Bcast:192.168.10.255  Mask:255.255.255.0
--
vmnet8    Link encap:Ethernet  HWaddr 00:50:56:C0:00:08  
          inet addr:192.168.12.1  Bcast:192.168.12.255  Mask:255.255.255.0

In this example, eth0 is the primary network interface. Ignore vmnet interfaces, they are created for VMware's Virtual Networking system.

Information for administrators

This error may also occur because you have improperly set the LAN_IFACE setting, or you are accessing it from a non-LAN address. In which case, go ahead and fix that.

It could also mean that there's a problem with tollgate trying to access the system's ARP table. Improper configuration of security extensions may cause this.
